1. Define part type / event name:

```ts
export const ARTIFACT_PART_TYPE = 'data-artifact'
```

2. Define part interface (This is DTO - Data Transfer Object)

```ts
export type ArtifactPart = {
  type: typeof ARTIFACT_PART_TYPE
  data: Artifact
}
```

3. Define event (will send part to stream)

```ts
export const artifactEvent = workflowEvent<ArtifactPart>()
```

4. Process workflow stream to transform events to SSE format (content-type: text/event-stream)

```ts
`data: JSON.stringify(part)\n\n`
```

5. Vercel AI SDK will parse the stream and convert event back to parts inside new assistant message.

```ts
const assistantMessage = {
  role: 'assistant',
  parts: [
    {
      type: "data-artifact",
      data: {...}, // artifact data
    },
  ],
}
```

6. In Chat UI, we already built a component (ArtifactPartUI) to able to render a Artifact UI from artifact parts data

```tsx
export function ArtifactPartUI({ className }: { className?: string }) {
  const part = usePart('data-artifact') // get artifact part
  if (!part) return null
  return <ArtifactCard data={part.data} className={className} /> // render an artifact card
}
```

7. Each artifact part in message.parts will be rendered by a ArtifactPartUI component. 
We can also use other built-in part components and custom part components.

```tsx
<ChatMessage message={message}>
  <ChatMessage.Content>
    {/* Built-in part components */}
    <ChatMessage.Part.Markdown />
    <ChatMessage.Part.File />
    <ChatMessage.Part.Event />
    <ChatMessage.Part.Artifact />
    <ChatMessage.Part.Source />
    <ChatMessage.Part.Suggestion />

    {/* Custom part components */}
    <WeatherPart />
    <WikiPart />
  </ChatMessage.Content>
</ChatMessage>
```
